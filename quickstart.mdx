---
title: "Get started with DNSHE Free Domains in minutes"
description: "Register a free DNSHE subdomain, add a DNS record, verify global resolution, and configure automated renewal in under five minutes."
---

Getting your first subdomain live on DNSHE takes only a few minutes. You will register an account, claim a subdomain on one of the four public suffixes, point it to your server, and confirm global resolution — all without entering a credit card.

<Steps>
  <Step title="Register at dnshe.com">
    Go to [dnshe.com](https://www.dnshe.com) and create a free account. No credit card is required. Once registered, log in to access the domain management console.
  </Step>

  <Step title="Search and claim a subdomain">
    In the console, search for your desired prefix. Choose a suffix that fits your use case:

    - `*.us.ci` — CI/CD pipelines, SaaS, API endpoints
    - `*.cc.cd` — Creative projects, portfolios
    - `*.de5.net` — Technical blogs, docs, dev environments
    - `*.ccwu.cc` — Personal pages, community sites

    For example, to claim `myapp.us.ci`, enter `myapp` and select the `us.ci` suffix. If the prefix is available, confirm the registration.
  </Step>

  <Step title="Add a DNS record">
    After claiming your subdomain, navigate to its DNS management page and add an A record to point it to your server's IP address.

    **Example — A record pointing to `203.0.113.10`:**

    | Type | Name | Content | TTL |
    | :--- | :--- | :--- | :--- |
    | A | myapp.us.ci | 203.0.113.10 | Auto |

    You can also create this record via the API:

    ```bash
    curl -X POST "https://api005.dnshe.com/index.php?m=domain_hub&endpoint=dns_records&action=create" \
      -H "X-API-Key: cfsd_xxxxxxxxxx" \
      -H "X-API-Secret: yyyyyyyyyyyy" \
      -H "Content-Type: application/json" \
      -d '{
        "subdomain_id": 1,
        "type": "A",
        "content": "203.0.113.10"
      }'
    ```

    <Note>
      DNS changes propagate globally in seconds thanks to DNSHE's Anycast infrastructure.
    </Note>
  </Step>

  <Step title="Verify it works">
    Once the record is saved, confirm resolution using `dig` or `nslookup`:

    ```bash
    dig myapp.us.ci
    ```

    ```bash
    nslookup myapp.us.ci
    ```

    You should see your IP address in the answer section. You can also verify via the API by listing your subdomains:

    ```bash
    curl -X GET "https://api005.dnshe.com/index.php?m=domain_hub&endpoint=subdomains&action=list" \
      -H "X-API-Key: cfsd_xxxxxxxxxx" \
      -H "X-API-Secret: yyyyyyyyyyyy"
    ```

    A successful response looks like this:

    ```json
    {
      "success": true,
      "count": 1,
      "subdomains": [
        {
          "id": 1,
          "subdomain": "myapp",
          "rootdomain": "us.ci",
          "full_domain": "myapp.us.ci",
          "status": "active"
        }
      ]
    }
    ```
  </Step>

  <Step title="Set up renewal">
    DNSHE provides a 180-day renewal window — the longest in the industry. You have two options:

    **Option A — Automated renewal via API:**

    ```bash
    curl -X POST "https://api005.dnshe.com/index.php?m=domain_hub&endpoint=subdomains&action=renew" \
      -H "X-API-Key: cfsd_xxxxxxxxxx" \
      -H "X-API-Secret: yyyyyyyyyyyy" \
      -H "Content-Type: application/json" \
      -d '{"subdomain_id": 1}'
    ```

    Schedule this command with a cron job or CI pipeline to renew automatically before expiry.

    **Option B — Calendar reminder:**

    Set a recurring reminder in your calendar to log in to the [DNSHE console](https://www.dnshe.com) and renew from the dashboard before your subdomain expires.
  </Step>
</Steps>
